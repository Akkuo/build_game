# 物理硬體與 UI 空間擴展字典 (UI Hardware Specs Reference)
> 本檔案記錄所有跨專案適用的行動裝置物理邊界、螢幕限制與防呆設計準則。
> **掛載時機 (On-demand Payload)**：當 AI 在進行盤面佈局規劃、關卡空間分析，或 UI 物件尺寸審查時，請呼叫此指南。

## 1. 基準解析度與安全區 (Safe Area)
- **標準豎屏邊界**：所有行動端休閒遊戲預設開發基準為 `1080(W) x 2340(H)`。
- **佈局限制**：所有的核心互動區域 (Gameplay Area) 必須容納在此幾何空間內，且底層規劃須保留底部 iOS Swipe Indicator (手勢小白條) 以及頂部的瀏海/動態島區塊的安全邊距。

## 2. 觸控物理防呆法則 (Touch Accessibility)
- **最低觸控寬度**：為符合人類正常手指點擊，單一可互動 UI 或場景物件的橫向受力空間（含間距 Margin/Padding）必須大於 `120px ~ 150px`。
- **空間切割極限**：在 1080px 的寬度極限下，單排水平佈局最多只能容納 `6~7 個目標物件`（例如：Magic Sort 的試管陣列）。
- **超載處置規範**：若企劃強行要求單排超過 7 個物件，必須強制實裝「二段式確認操作（點起預覽 -> 點擊確認）」或提供「放大鏡/螢幕外輔助定位標籤」。否則，這在工程標準上將被視為會引發極高誤觸流失率的不可用 (Invalid) 設計。
