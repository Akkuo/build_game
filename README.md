# 🚀 Casual Game Teardown Framework (休閒遊戲拆解分析框架)

這是一個專為休閒遊戲（如：混合式解謎、挪車、合成、三消等）打造的**自動化分析與逆向工程框架**。

本專案整合了 AI Agent 人格設定、拆解標準作業流程（SOP），以及對齊開發團隊實務運作的分析產出規格，旨在讓競品分析不再只是主觀感受，而是能直接轉換為專案立項與開發規格書的量化數據。

## 📁 核心架構

本專案包含以下三大核心模組：

### 1. 🤖 專家 Agent 人格設定 (Persona Definitions)
我們定義了五位具備專業職能的 AI 助手，協助在不同維度上深度剖析遊戲：
- **`game-designer.md` (遊戲設計師)**：負責核心循環、數值體系、防呆機制與商業化拆解。
- **`level-designer.md` (關卡設計師)**：負責難度曲線、空間佈局與節奏架構分析。
- **`game-audio-engineer.md` (音效工程師)**：負責 ASMR 效應、聽覺回饋迴圈與記憶體預算評估。
- **`technical-artist.md` (技術美術)**：負責目標機型的渲染效能、LOD、VFX 管線與資源規格估算。
- **`narrative-designer.md` (敘事設計師)**：負責 Meta 遊戲主題包裝、輕故事與情感誘因拆解。

### 2. 📋 拆解標準作業流程 (Teardown SOP)
- **`teardown-plan.md` (逆向拆解計畫書)**：從 0-30 秒的即時反饋到數天的長期留存鉤子，提供五大維度、步驟明確的拆解 Guideline。

### 3. 🎯 產出對齊規格 (Deliverables Spec)
- **`teardown-deliverables-spec.md` (拆解產出規格書)**：規範拆解後的所有數據與結論，必須完全對齊團隊內部的「規格書 Excel 範本」（如 `FarmJam_規格書.xlsx`），確保拆解報告能無縫接軌至後續的「製作項目清單」與「外包驗收標準」。

## 🛠️ 使用方式

1. 啟動 AI 工作環境並指派對應的 Agent 人格（例如：在對話中呼叫/參照對應的 `.md` 設定）。
2. 依照 `teardown-plan.md` 的指示，針對目標休閒競品進行切片拆解。
3. 將產出的數據、循環圖與系統剖析，依照 `teardown-deliverables-spec.md` 的要求，填寫入團隊標準的 Excel 規格書中。

## 📄 維護與版本控制
所有流程指引與 Agent 檔案皆使用 Markdown 撰寫並受 Git 版本控制。團隊可依據未來的專案變更（例如重心轉移至不同品類），隨時於對應的 `.md` 檔案中新增「修訂記錄」並調整分析基準。
