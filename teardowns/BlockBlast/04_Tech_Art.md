# Block Blast! 規格書 - 04. 效能與美術拆解
> 分析基礎：極致 Unlit 烘焙渲染、數學補間與零發熱耗電
> 負責人：Technical Artist AI

## 1. 視覺欺騙與渲染管線 (Rendering Pipeline)
### 1.1 偽 3D 的純 2D 烘焙 (Pre-baked 2.5D)
- **絕對 0 光照**：雖然玩家肉眼看到的方塊，邊緣帶有高光 (Highlight) 且底部帶有倒角陰影 (Drop Shadow)，呈現出圓潤的立體感；但這款遊戲**底層完全捨棄了 3D 模型與光照引擎**。
- **UI 切圖 (9-Slicing)**：所有的方塊實質上就是單純的 2D 圖片，透過九宮格切片 (9-Slice Sprite) 避免變形。所有的光影都是美術直接「烘焙 / 畫死 (Baked)」在像素圖層上的。這個決策 100% 解放了 GPU 的即時動態光影算力。

### 1.2 Overdraw 絕對下限控制 (Zero Overdraw)
- **極致扁平化**：遊戲背景是一張單純的靜態單色漸層底圖，8x8 盤面網格是一層 UI，方塊是一層 UI。整片手機螢幕的圖層過度繪製 (Overdraw) 重疊次數精準維持在物理極限的 `1.0 ~ 1.5` 層之間。
- **不發燙的留存神話**：因為徹底沒有半透明物理疊加 (Alpha Blending)、全螢幕後處理 (Post-processing) 或是即時陰影算繪，Block Blast 成為了真正的「護機神作」。在極舊的低規設備 (iPhone 6 或是破舊的安卓機) 上連玩數小時也不發熱、極度省電。這個 TA 決策直接保障了「無盡殺時間遊戲」所需的超長線上留存時間 (Session length)。

## 2. 擠壓手感與動態反饋 (Juiciness & Tweening)
*Block Blast 毫骨骼動畫，它刀刀見骨的打擊感，全部來自於極高頻率的純數學補間演算 (Math Tweening)。*

### 2.1 UI 的微彈性碰撞 (Elastic Scaling)
- **拾起動畫 (Pick-up)**：手指壓到候選方塊的瞬間，方塊會套用一個極短時間的 `EaseOutBack` (過衝彈回) 動畫，在 0.1 秒內放大到 1.15 倍並微幅超出邊界後，抖動彈回到 1.1 倍穩定懸停。這創造了物體「帶著黏性被連根拔起」的物理重力張力。
- **觸地落實 (Drop & Thud)**：當方塊成功塞入盤面的網格時，模型會瞬間在 Y 軸上極度收縮至 0.95 倍再反彈回 1.0 倍正常原樣。雖然視覺上僅有幾 Pixels 的些微抖動差異，但這是配合第三步拆解的「實心木塊敲擊音效」所構成的完美視聽聯覺 (Synesthesia) 打擊點。

### 2.2 破碎粒子的遮罩運算 (Shatter Illusion)
- **假碎裂特效**：當整排被消除時，並不是真正讓方塊碎成 3D 幾何破片。系統只在此排方塊的最上層，瞬間疊加暴力的快閃高亮白光 (Flash Mask)，同時從中心點爆破一組極度簡單的 2D 對稱碎片粒子特效 (Particle System)。
- 接著底下的原實體方塊只不過是執行了 `Scale: 1.0 -> 0.0` 的迅速縮放至消失。用最低等的計算資源，換取了破壞狂視角下極為華麗的碎裂演出。

## 3. 資源加載與合批策略 (Draw Call Batching)
### 3.1 極致的合批控制 (Draw Calls Limitation)
- 因為盤面上的所有彩色方塊皆不帶有獨立 Shader，不依賴半透明穿透，這意味著它們可以被輕易且完美地打包進同一張圖集 (Texture Atlas) 當中。
- 在底層引擎中執行時，哪怕 64 個格子全部塞滿不同顏色的巨型方塊，整個遊戲的 Draw Call 次數也能死硬地壓在 個位數 (Singledigit) 以內。這保障了遊戲在任何極端堆疊操作下，都能穩如磐石地維持 60 FPS，不讓一絲絲的卡頓(Spike)打斷玩家上癮的節奏。
