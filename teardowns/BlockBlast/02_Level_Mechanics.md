# Block Blast! 規格書 - 02. 關卡與空間分析
> 分析基礎：無盡模式單局節奏拆解
> 負責人：Level Designer AI

## 1. 空間佈局與防呆視覺設計
### 1.1 盤面長寬與熱區 (Heatmap) 估解
- **8x8 極簡盤面**：總計 64 個網格。選擇 8x8 (而非傳統的 10x10) 刻意縮小了生存空間極限，大幅加快單局死亡循環，藉此增高廣告曝光頻率。
- **視覺熱區轉移**：
  - **操作舒適區（底部候選區）**：佔據螢幕下半部約 25% 空間，為單手握持拇指最舒適的掃掠區域，不需頻繁更換手勢。
  - **死亡盲區 (Blind Zone)**：盤面的最底下兩列。因為玩家手指拖放方塊時多在螢幕中下方移動，容易與掌根產生視覺遮擋。因此系統在設計上刻意引導，讓玩家傾向將大型方塊往「盤面上半部」推砌。

### 1.2 無文字空間引導 (Non-verbal Onboarding)
- **陰影預判 (Projection Shadow)**：當玩家拖曳方塊懸停在盤面上方時，正下方會立刻投射對應形狀的半透明暗色高光，讓玩家在尚未鬆手的情況下，百分之百確認落點。這是減少「非戰之罪（滑動導致誤放）」而流失的關鍵防護機制。
- **邊緣與死區提示 (Edge Warning)**：方塊靠近無法放置的邊界或重疊區域時，預判陰影不會生成，直接不予顯示。此視覺反饋以極低認知負擔的方式，強制教導玩家空間不足的概念。

## 2. 難度曲線與節奏譜分析 (Pacing)
*由於 Block Blast 為無盡挑戰，無關卡之分，故將單局體驗以「分數閾值（存活時間）」劃分為三個階段的高低壓光譜，藉此對齊 `關卡資料` 的節奏要求。*

### 2.1 階段一：教導與建立自信 (0~800分)
- **節奏特徵**：極低壓。
- **空間給定狀態**：整個 8x8 盤面（64格）為幾乎淨空。
- **系統發牌 (RNG) 控制**：嚴格限制產出，最高只會掉落面積 ≤ 4 格的模塊（如 2x2 正方體、1x3 橫/直線、短 L 型）。絕對不發大型 3x3 模塊。
- **設計目的**：讓玩家瞎放也能隨便消去行/列，累積微量自信與基礎手感。

### 2.2 階段二：爽快爆發與應用期 (800~3000分)
- **節奏特徵**：心流區間（正向回饋頻率極大化）。
- **空間給定狀態**：盤面常態會保持在 30%~60% 左右的佔用率（19~38格）。
- **系統發牌 (RNG) 控制**：開始解鎖 5 連長條與各類複合大型模塊。
- **設計目的**：此區間玩家已經掌握「要故意留 5 格長條空位」、「要注意邊角縫隙」的空間佈局觀念。遊戲開始給足長條或關鍵色塊，強力誘導玩家進行連消（Combo），進入本遊戲最容易上癮的爆發區間。

### 2.3 階段三：高壓逼迫與變現卡關 (3000分以上)
- **節奏特徵**：壓力頂峰（逼近 Game Over）。
- **空間給定狀態**：盤面常態佔用率突破 75%（超過 48 格被佔據），整張圖只剩下零星破碎的無效空間。
- **系統發牌 (RNG) 控制**：系統無情進入「殺戮模式」。每次發的 3 個模塊中，必定配一個大面積的 `3x3` 正方形或是 `十字型` 極度卡位的模塊。只要盤面上沒有預先留好這 9 格巨大洞穴，保證一擊斃命。
- **設計目的**：強制閹割單局時間。這不是玩家技術差，而是數學上網格已經物理性無法容納。這會強硬逼出玩家「我剛才差一步就能消掉那一大排」的不甘心，進而點擊按鈕去貢獻廣告營收。

## 3. 破局與逆轉空間配置
### 3.1 局內道具機制的捨棄
- 與同類競品（如 Woodoku 或 Tetris）不同，Block Blast 刻意**捨棄了單局內常駐的破冰操作**（例如旋轉模塊、洗牌、炸彈）。
- **設計考量**：不給予玩家主動改變或破壞空間配置的能力，能更完美且獨裁地控制單局生命週期。玩家只能完全臣服於系統的發牌邏輯。

### 3.2 復活機制的空間讓利
- 遊戲將唯一的「空間破除炸彈」綁定在觸發 Game Over 時的「觀看廣告復活」。
- **清場讓利邏輯**：玩家看完廣告後，系統會自動鎖定盤面上**最密集、最妨礙放置的核心 3x3 或 4x4 區域**直接爆破移除。這剛好騰出了一把巨大的空間鑰匙，讓玩家瞬間從高壓（階段三）跌回前述的教導舒適區（階段一/二），並保住好不容易累積的連擊與分數。
