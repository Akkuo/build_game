# Block Blast 規格書 - 02. 關卡與核心機制
> 分析基礎：實體觸控防呆與 UI 絕對安全局限
> 負責人：Level Designer AI

## 1. 物理極限約束區劃 (Hardware Physical Bounds)
### 1.1 黃金防誤觸的 8x8 矩陣 (The Ergonomic 8x8 Grid)
在行動裝置強制基準 `1080(W) x 2340(H)` 的豎屏極限中，8x8 是經歷精密計算的最佳解。如果改成 10x10，網格的絕對像素尺寸會跌落到大拇指的遮蔽半徑內，導致極高的放置失誤率；若是 6x6，則策略深度過於簡陋。8x8 確保了單一 Grid 維持在約 120px，完美吻合人體工學的點擊區域。

### 1.2 底部隔離配發槽 (Deck Safe Zone)
為防阻玩家操作牌組時誤觸螢幕底部的 Home 橫條 (iOS Home Indicator) 或是 Android 虛擬導航鍵，盤面正下方被規劃出了一塊高度絕對鎖定的空白配發區（約 400px）。這三個方塊模型必須等距散佈於此區域，且在該區域的任何操作都不會引發全域畫面的誤滑。

## 2. 懸浮操作與手感工程 (Ghost Drop Interaction)
### 2.1 Y軸強制偏移防禦 (Y-Axis Offset Compensation)
觸控螢幕遊戲的核心痛點：「手指按住模型時，會百分之百擋住想要辨識的網格」。本機制在判定手指按下的瞬間，實體圖像不僅要微縮，還必須強力向 Y 軸方向「上移 150px」。這樣方塊本體就彷彿被吊掛在玩家指尖的前方，視線得以繞過手指，無阻礙地確認方塊對接的戰況。

### 2.2 磁吸感應與底層發光 (Matrix Snapping Indicator)
配合鬼影懸浮機制，系統必須每幀運算該方塊最左上角的座標坐落於哪個網格基準。當方塊有任一角落通過合法的空缺處時，該對應空位底部必須亮起具備呼吸燈特效的半透明底色，這能直接向玩家暗示：「放開手，我就會完美切入這區。」

## 3. 非毀滅性操作懲罰 (Non-Destructive Penalty)
### 3.1 違法碰撞的紅光拒止 (Collision Red-light)
如果拖曳的方塊模型有一部分重疊了已被填滿的網塊，或是超出 8x8 烤盤，預覽底部將會轉為嚴厲的紅光。此時若釋放手指，不會觸發任何導致遊戲結束的破壞行為或直接死亡。

### 3.2 彈力繩退回 (Elastic Snap-back)
當玩家在紅色違法區或盤面外釋放方塊時，方塊將具備彈性係數 (Elastic physics)，在 0.3 秒內以拋物線自動被拉回下方配發槽的原始座標。這種做法提供了必要的「操作碰壁感」，但因其極度滑順的動畫表現，反倒減少了焦躁感，讓玩家可以沒有任何顧忌地滑動手指瘋狂試誤。
