# Magic Sort 規格書 - 04. 效能與美術拆解
> 分析基礎：假流體渲染、Overdraw 控制與擠壓動畫
> 負責人：Technical Artist AI

## 1. 核心渲染材質拆解 (Shader & Materials)
### 1.1 的假流體物理 (Fake Fluid Simulation)
水管分類 (Magic Sort) 必須涵蓋全球廣泛受眾，因此能讓最廉價的 Android 手機跑滿 60FPS 是鐵則。它**絕對沒有使用真實的流體物理引擎 (No Real Physics)**。
- **液體構成**：管內的液體段落，本質上只是互相堆疊疊加的四方形純色塊模型 (Opaque Polygons)。
- **表面張力動畫**：玩家所看到最頂層水面波光粼粼的搖晃感，成本極低。多數運用簡單的 Vertex Shader (頂點著色器)，只在頂端邊緣的頂點座標上掛載一組 Sine Wave (正弦波函數)，隨時間產生左右/高低的數學晃動，模擬出逼真的液態感。
- **水柱假象**：傾倒液體時連接兩管的水柱，不是真實算繪的水流，而是一張被拉伸的帶有高光的帶狀貼圖 (Sprite)，配上不斷向下高速捲動的材質 UV (UV Scrolling)，並在落水點加上簡單的 2D 水花粒子。

### 1.2 玻璃與光影的極簡化 (Cheap Glass Material)
- **拒絕即時折射 (No Real-time Refraction)**：真實玻璃的折射、焦散在移動底層極其耗能。此類遊戲中的玻璃管多採用**純透明底圖 + 事先畫死的烘焙高光 (Baked Highlight)**。
- **嚴謹的三明治圖層結構**：
  1. `底層 (Z=0)`：瓶子的背部暗色內框邊緣。
  2. `中層 (Z=1)`：完全不透明 (Opaque) 的各色液體方塊。
  3. `頂層 (Z=2)`：帶有 Alpha 透明度、畫著純白玻璃高光與反光的瓶身前壁。
- 這種偷吃步的結構，創造了晶瑩剔透的錯覺，卻讓單一試管的 Shader 計算量幾乎等同於毫無光照的最低階物件 (Unlit)。

## 2. 記憶體與特效預算 (Memory & VFX Budgets)
### 2.1 Overdraw (過度繪製) 避險限制
- 當多根管子與背景交疊時，若使用半透明材質會產生嚴重的效能災難 (Overdraw)。
- 系統的強制解法：**液體絕對不可使用半透明 (Alpha Blending)**，而是 100% 寫入深度的實心色塊。另外，遊戲背景多為毫無層次的單張靜態 2D 圖，將整部手機螢幕的 Overdraw 重疊次數嚴格封殺在 2~3 層之內，防止硬體發燙降頻。

### 2.2 完全物件池管控 (Strict Object Pooling)
- **過關爆破**：純色歸位時瓶口炸開的「星星、五彩紙屑」，粒子落到畫面中段就會強制 Alpha 淡出消失，絕對不讓粒子堆積或是開啟底層物理碰撞。
- **無動態生成 (Zero Instantiate)**：所有的彩色液體區塊、爆破特效粒子、甚至用來復活的「+1 空瓶」，在玩家按 Start 的那一刻就全數在記憶體載入完畢並隱藏 (SetInactive)。遊戲過程中的水位升降、倒水，只是透過改變 Transform.scale (Y軸拉伸) 來調度，杜絕遊戲中途因分配記憶體而產生的微卡頓 (Spike)。

## 3. UI 反饋與動態手感 (Juiciness/Game Feel)
### 3.1 數學形變反饋 (Squash & Stretch)
- **提瓶浮空**：當玩家點擊水管使其升起時，水管並非只是死板地往 Y 軸移動。引擎會賦予其在 X 軸微幅縮小再彈回的貝茲曲線動畫 (Scale X: 1.0 -> 0.93 -> 1.04 -> 1.0)。這賦予了原本 2D 的靜態圖片如同真實玻璃瓶被「拔起」的沉重手感。
- **拒絕錯位回彈**：若點擊錯誤的目標管（顏色不同或滿載），水管飛到一半會光速彈回原始座標，並強制伴隨一組極端快速衰減的「左右正弦位移 (Shaking)」，透過這個物理性的搖頭晃腦，完美取代文字去告訴玩家「此路不通」。
