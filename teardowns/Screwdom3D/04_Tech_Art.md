# Screwdom 3D 規格書 - 04. 效能與美術拆解
> 分析基礎：Low-poly 3D 渲染策略、GPU Instancing、PBR Lite 金屬材質
> 負責人：Technical Artist AI

## 1. 核心渲染材質拆解 (Shader & Materials)

### 1.1 金屬螺絲的 PBR Lite 實作
Screwdom 3D 的核心視覺賣點在於「真實金屬螺絲感」，但行動端無法負擔全套 PBR 管線的速算成本。其實作策略如下：
- **Metallic 值**：設定為 0.9（高金屬感），Roughness 為 0.2（低粗糙度 = 高光澤）。
- **Normal Map（法線貼圖）**：僅在螺絲頭貼圖加入六角螺帽的細節法線貼圖（128x128px），在不增加幾何面數的前提下，讓光線在螺絲頭表面產生六面菱角的反光效果。
- **Specular 高光貼圖**：螺絲本體使用事先烘焙（Baked）的高光貼圖而非即時計算，渲染成本等同 Unlit（無光照）Shader。

### 1.2 3D 模型的 Low-poly 效能策略
- **低多邊形幾何（Low-poly Mesh）**：所有 3D 模型的多邊形面數控制在 500~2000 個三角面以內（Tri Count），確保全機型 60fps。
- **三段 LOD（Level of Detail）**：
  - LOD 0（近景 / 操作中）：完整面數模型 + Normal Map
  - LOD 1（遠景 / 結算動畫）：面數降低 50%，無 Normal Map
  - LOD 2（縮圖預覽 / 關卡選擇）：面數降低 80%，純色平坦著色
- **GPU Instancing**：同一 3D 模型上的多個相同顏色螺絲，使用 GPU Instancing 批次繪製，將 Draw Call 從「每顆螺絲一次」壓縮至「每種顏色只需一次」。

### 1.3 顏色收集盒的假 3D 策略
- 收集盒視覺上看起來是 3D 容器，實際上可能只是：
  1. 一個實心 3D 盒子 Mesh（底面 + 四側面，無頂蓋）
  2. 盒子內部使用純色填充的 Quad，模擬積累的螺絲堆疊視覺，**不生成真實的 3D 螺絲物件**（用貼圖欺騙即可）
- 此設計在收集盒已收入大量螺絲時，完全避免了場景內 3D 物件數量爆炸導致的效能危機。

## 2. 記憶體與特效預算 (Memory & VFX Budgets)

### 2.1 Overdraw 避險限制
- **背景環境**：採用單張靜態 HDRI 烘焙底圖或純色背景，將底層 Overdraw 鎖死在 1 層。
- **螺絲飛行拖尾**（若有）：拖尾特效粒子在離開螢幕前強制 Alpha 淡出消失，禁止粒子觸及底層進行混合計算，確保 Overdraw 全場景 ≤ 3 層。
- **透明材質禁令**：3D 模型的主體 Mesh 全部使用不透明 (Opaque) Shader，透明材質只允許用在極少數的高光前壁貼圖上，且面積嚴格控制。

### 2.2 Object Pooling（物件池管控）
- **螺絲物件池**：遊戲開始時預先在記憶體中初始化所有螺絲物件（SetActive = false），過關 / 重置時只重置 Transform 與材質色彩，絕對不在遊戲中途動態生成（Instantiate）或銷毀（Destroy）任何螺絲。
- **特效粒子池**：入盒爆散星星、過關崩解粒子均以固定數量預先在場景中待命（max 200 個粒子），超量時自動回收最舊粒子，防止記憶體微卡頓（GC Spike）。

## 3. UI 反饋與動態手感 (Game Feel)

### 3.1 螺絲旋轉的數學形變反饋
- **取下的螺旋路徑**：螺絲移除時不只是直線上升，而是沿 Pin 的螺旋軸旋轉（Rotation + Y-axis Move Tween），模擬真實物理鬆螺絲的動作，給玩家強烈的「我真的把它轉下來」的實際觸感。
- **拋物線飛行**：螺絲從 3D 模型飛入收集盒的路徑採用貝茲曲線（Bezier Curve），配合加速衰減（Ease In-Out），視覺上自然流暢。

### 3.2 Squash & Stretch（入盒形變）
- 螺絲飛入收集盒的瞬間，物件在 Y 軸微幅壓扁（Scale Y: 1.0 → 0.8 → 1.1 → 1.0），模擬金屬物件碰到硬壁後的彈性形變，強化「有重量有實體感」的觸覺代入。

### 3.3 Screw Jam 拒絕反饋
- 被卡死的螺絲點擊後執行快速正弦位移 Shake（水平 X 軸，振幅約 ±8px，頻率 3 次 / 0.2 秒），配合高亮紅色邊框閃爍，傳達「此路不通」的明確視覺語言，無需任何彈出文字提示。

## 4. 技術效能指標（目標值）

```
目標幀率：              60fps（全機型）
最高 Draw Call 數：     ≤ 50 次 / 幀
最高 Tri Count：        ≤ 50,000 個三角面 / 幀
Overdraw 層數上限：     ≤ 3 層
貼圖記憶體預算：        ≤ 256MB（含所有 Sprite Atlas 與 3D 貼圖）
GC Allocation / 幀：    0（零動態記憶體分配）
啟動載入時間：          ≤ 3 秒（所有物件預初始化完畢）
```
