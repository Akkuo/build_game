## Tab 1 - 遊戲規則 (Game Rules)

### 核心操作定義
- **操作模式**：單指點擊拆解 (Tap & Collect)。玩家點擊 3D 模型上的螺絲，螺絲自動旋轉鬆開並「飛入」對應的顏色收集盒（Color-coded Box）。不採用拖曳操作，降低誤觸門檻。
- **旋轉盤面 (Dynamic Camera)**：玩家可以用一指滑動旋轉整個 3D 模型，觀察被遮擋的螺絲位置。這是對「Z 軸視覺死角」問題的核心解法，同時增加探索樂趣。

### 物理容積限制
- 每個顏色收集盒有固定容量（通常 6~10 顆螺絲）。若目標收集盒已滿，即使螺絲顏色正確，該螺絲也無法取下（等同 Magic Sort 的「容積滿載規則」）。
- **Pin 結構限制**：某根螺絲若被其他結構件「壓住」，即使想取該螺絲，系統也不允許（物理卡住 = Screw Jam）。

### 轉移規則（Screw Jam 核心規則）
- 每顆螺絲只能放入顏色相同的收集盒。
- 螺絲取下時，系統判斷該螺絲下方的 Pin 是否仍有其他結構件依附，若有，那些結構件會產生「位移下垂」的物理反饋動畫（偽物理模擬）。
- **死局 = Screw Jam**：當盒子全滿、所有可見螺絲又因為 Pin 結構被卡死無法取下，即觸發死局。

### 勝敗條件
- **勝利**：3D 模型上的所有螺絲全數取下，結構件全部分離落下，顯示過關動畫。
- **失敗 / 死局**：當所有可以操作的螺絲均形成 Screw Jam、且收集盒已無剩餘容量，無法繼續進行。

### 容錯設計
- **Undo（上一步）**：每局提供 3 次免費額度；耗盡後須看廣告或花費金幣。
- **Extra Box（擴充收集盒）**：支付廣告或金幣，臨時新增一個中性顏色的緩衝收集盒（最強道具，破壞力等同 Magic Sort 的 +1 空瓶）。
- **無時間限制**：明確定位為無壓力解謎，無計時機制。

### 操作極限邊界
- **Hitbox 設計**：螺絲頭在 3D 空間的可點擊區域需膨脹至視覺面積的約 140%，補償手機觸控的 Z 軸深度誤判（3D 遊戲比 2D 更難精準點選）。
- **旋轉慣性**：Camera 旋轉應有輕微的 Momentum（手放開後繼續轉一點點再停），增加 3D 空間感的真實觸感。

## Tab 2 - 主題包裝 (Theme & Meta Story)

### 世界觀設定
- **表層主題**：機械拆解 / 工程師美學。螺絲釘的實體感與撬開結構的「破壞式成就感」是核心情緒錨點。
- **皮相彈性 (Reskinning)**：已知市面上有「房屋拆解」、「動物公仔」、「交通工具」等多套皮相，核心算法不變，靠皮相置換擴大受眾。

### 情感鉤子 (Emotional Hook)
- **拆解控制欲**：螺絲一顆顆被取下，結構件逐漸分離，最後完全崩解的完成感，直接命中「拆卸、清理、整理」的 OCD 式多巴胺迴路。
- **空間解謎的智識優越感**：需要提前規劃「哪顆先拆」才不會卡死，比 Magic Sort 的純色彩匹配多了一層空間邏輯思考，滿足輕度燒腦的玩家。

### 挫折敘事設計
- **Screw Jam 懊悔時刻**：當所有操作路徑被自己拆錯而封死時，系統不立刻 Game Over，而是讓玩家在無解的 3D 模型前「乾瞪眼」，視覺上明顯看到哪顆螺絲卡在哪裡導致全盤皆輸，觸發強烈懊悔感。
- **Pity System（差一格）**：難度曲線上，系統刻意讓 70% 的卡關都是「只要再多一個收集盒空間」就能解決，是最低成本的廣告轉換誘餌。

### Meta 層（暫無強 Meta）
- 目前以純關卡通關制為主；部分版本加入「解鎖新 3D 模型主題」作為輕量 Meta 動力，尚未達到 Magic Sort 的 Home Renovation 等級的長線敘事。

## Tab 3 - 流程圖 (Game Flow)

[流程] 啟動遊戲 / 選關 -> 進入 3D 盤面（模型旋轉展示動畫）-> 玩家觀察並點擊螺絲
[判斷] 螺絲可取下？無 Screw Jam？
  - 否（結構被卡住）-> 螺絲震動提示，無法取下音效 -> 回到點擊螺絲
  - 是 -> 螺絲旋轉動畫，飛入顏色收集盒
[判斷] 收集盒顏色配對且有剩餘容量？
  - 否（盒子滿或顏色不符）-> 螺絲飛回原位，錯誤音效
  - 是 -> 螺絲入盒，ASMR 點擊音效
[判斷] 模型所有螺絲已全數移除？
  - 否 -> 形成 Screw Jam 死局？
    - 否 -> 繼續點擊
    - 是 -> 盤面凍結，高亮卡死路徑 -> 使用道具？
      - 是 -> 繼續
      - 否（放棄）-> 插屏廣告 -> 重新選關
  - 是（勝利）-> 全模型崩解特效 -> 星星評分結算 -> 插屏廣告 -> 選關畫面

## Tab 4 - 遊戲介面 (UI Layout)

### 主要面板（以 1080x2340 為基準座標系）
頂部 HUD（Y: 0~180px）：關卡編號、設定、進度條
3D 模型區（Y: 180~1600px）：核心旋轉盤面，佔約 60% 螢幕高度（主操作區）
收集盒區（Y: 1600~2000px）：顏色分類收集盒排列
道具欄（Y: 2000~2200px）：Undo、Extra Box、Hint 快捷鈕
安全區（Y: 2200~2340px）：保留，避免手勢列衝突

### 觸控熱區 (Fitts's Law)
- 道具欄落在 Y 軸下方 1/4，大拇指可直接觸及。
- 收集盒故意做大（對比螺絲頭），視覺引導玩家注意「還有多少空間」。
- 3D 旋轉手勢區：整個 3D 模型區均可滑動旋轉，無需專屬旋轉鈕，降低 UI 複雜度。

### HUD 資訊密度
- **常駐顯示**：關卡編號、已使用 Undo 次數、剩餘收集盒空間。
- **隱藏不顯示**：DDA 動態難度參數、廣告計數器。

### 彈出層優先級
1. 死局道具廣告彈窗（最高層）
2. 過關結算面板
3. 購買道具金幣確認框
4. 設定選單

## Tab 5 - 美術靜態開圖 (Static Art Specs)

素材名稱 / 類型 / 建議輸出尺寸 / 備註
- 3D 主模型：低多邊形 Mesh (Low-poly)，動態 LOD，顏色鮮豔高對比，確保螺絲辨識度
- 螺絲頭 Mesh：3D 模型 + 金屬材質貼圖，128x128px 貼圖 @3x，六角螺帽標準造型配金屬 Normal Map
- 顏色收集盒：3D 模型或 2D 假 3D，256x256px @2x，底部固定，顏色高飽和易辨識
- 背景環境：靜態 HDRI 或單張底圖，2048x2048px，柔和工作室燈光感，不搶主視覺
- UI 按鈕圖示：Sprite Atlas，128x128px @3x，合批入同一圖集

## Tab 6 - 美術動態開圖 (Animation Specs)

動畫名稱 / 觸發條件 / 技術實作 / 時長
- 螺絲旋轉鬆開：點擊合法螺絲，沿螺絲軸旋轉 + Y 軸向上位移 Tween，0.3~0.5s
- 螺絲飛入收集盒：取下後，拋物線路徑 Lerp + 縮放至盒子大小，0.4s
- 螺絲卡死震動：點擊 Screw Jam 螺絲，快速正弦位移 Shake（水平軸），0.2s
- 結構件位移下垂：依附的 Pin 被移除，偽物理 Tween (Y 軸輕微下移 + 晃動)，0.3s
- 全模型崩解：過關觸發，所有 Mesh 向外爆散 + 旋轉消失，1.0~1.5s
- 3D 模型入場展示：關卡開始，緩慢旋轉一圈後停止等待輸入，2.0s

幀率保障：目標 60fps，3D Mesh 全部使用 GPU Instancing 批次繪製。Overdraw 控制：背景使用不透明材質；螺絲飛行路徑的拖尾特效強制在離開螢幕前 Alpha 淡出。

## Tab 7 - 道具功能 (Item / Power-up System)

道具名稱 / 取得方式 / 功能描述 / 使用限制 / 成本
- Undo 上一步：免費 / 廣告，把最後取下的螺絲放回原位，每局 3 次免費，耗盡後看廣告
- Extra Box 擴充盒：看廣告 / 金幣，新增一個萬用緩衝收集盒，每局 2 次上限，30 秒激勵廣告（最強道具）
- Hint 提示：廣告 / 金幣，高亮顯示「下一顆最優先拆的螺絲」，每局 3 次，15 秒廣告
- Remove 直接清除：高級 IAP，直接移除指定一顆卡住的螺絲（無視規則），每局 1 次，高單價 IAP

## Tab 8 - 機關物件 (Obstacle & Special Objects)

機關名稱 / 出現條件 / 效果 / 解除方式
- 標準 Pin 螺絲：所有關卡，基礎操作單位可自由取下，直接點擊
- 結構依附件 (Bracket)：中高關卡，螺絲移除後部件下垂可能擋住其他螺絲，規劃正確拆解順序
- 鏽蝕螺絲 (Rusted)：推測高關卡，需特定道具才能鬆開，Remove 道具
- 隱藏螺絲 (Hidden)：中關卡起，藏在模型背面或內部需旋轉鏡頭，旋轉 3D 視角

DDA 連動說明：
- 前 20 關：保證初始視角可看到所有螺絲，無隱藏設計
- 21~60 關：開始引入依附件下垂機制，首次觸發視角死角
- 60 關以上：大量隱藏螺絲 + 依附件組合拳，自然難度飆升，IAP 轉換期

## Tab 9 - 美術風格資料 (Art Style Guide)

### 整體風格定調
明亮卡通 Low-poly 3D：高飽和、高對比配色，模型稜角清晰，螺絲色彩與結構件底色刻意拉開對比度，確保觸控識別無誤差。

### 主色盤 (Color Palette)
- 背景環境光：柔白工作室感，建議 HEX #F5F0EA
- 螺絲色 - 紅：鮮豔正紅，建議 HEX #E83535
- 螺絲色 - 藍：清透皇家藍，建議 HEX #3A7AE8
- 螺絲色 - 黃：明亮金黃，建議 HEX #F5C518
- 螺絲色 - 綠：翠綠，建議 HEX #3ACA6A
- 螺絲金屬感：鉻銀色調，#C0C0C0 + Normal Map 高光
- 卡住警告提示：使用時高亮，建議 HEX #FF4500

### 光影系統
Pre-baked 環境光 + 即時點光源（僅螺絲取下瞬間）：3D 環境光採烘焙，平衡效能與質感。螺絲飛出時觸發一個短暫的點光源 Flash，增強金屬高光感。金屬材質採用 PBR Lite，Metallic + Roughness 兩參數控制，在 Unity 或 Unreal 移動端 Shader 下均可高效運行。

### 字體規格
- 關卡標號 / 計分：粗圓無襯線字體，黑色描邊白色填充，可疊加於任何背景
- 道具說明：Regular Weight，深色系底板白色字

### 渲染策略
Shader：PBR Lite (Mobile)；螺絲旋轉動畫使用 GPU Skinning。材質壓縮：iOS ASTC 4x4；Android ETC2。LOD 策略：主模型分 3 個 LOD 等級，遠景切換至極低多邊形，避免複雜場景下的過熱。

## Tab 10 - 音樂音效需求 (Audio Requirements)

### 背景音樂 (BGM)
- **風格**：輕快工業機械感，搭配悠閒的木琴或鋼片琴旋律（避免太重的電子感，保持放鬆氛圍）
- **BPM**：約 100~120 BPM
- **循環點**：第 8 或第 16 拍換段，確保無縫 Loop
- **情緒層次**：平日輕快 -> 近死局時添加輕微張力弦樂 -> 過關爆發短旋律

### 核心音效 (SFX) 清單
事件 / 音效描述 / Pitch Shifting / 備註
- 螺絲旋轉鬆開：金屬「咔噠」機械聲，無，ASMR 核心，高度擬真金屬感
- 螺絲飛入收集盒：金屬輕碰「叮」，每連續入盒 +0.3 音階，多盒連續收集時形成音階爬升
- Screw Jam 卡死：低沉「嗯嗯嗯」金屬受阻聲，無，配合螺絲震動動畫
- 結構件下垂位移：輕微機械吱嘎聲，無，強化偽物理感
- 全模型過關崩解：爽快金屬彈飛散落聲，無，滿足感爆發點
- 使用 Undo：快速倒退音效，無，清晰的撤回感
- 使用 Extra Box：清脆出現音效，無，正向情緒強化
- 死局觸發凍結：低沉懊悔音 + 靜止，無，配合 2 秒凍結卡面

### 記憶體預算
- 預估音訊資源總量：小於等於 18MB（3D 遊戲的金屬音效 wav 品質要求較高）
- 壓縮格式：BGM 採 AAC 128kbps；SFX 全部 OGG Vorbis Q7（金屬音頻需保留更多高頻泛音）
- Audio Pooling：同一時間最多 8 個音效同時播放，超過自動 Cull 最舊的聲源，防止音頻堆疊爆音
