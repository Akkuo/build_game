# 01. 系統與經濟拆解 (調用 Game Designer)

## 🎯 拆解目的
拆出競品「為什麼讓人一直想玩」的驅動力，以及「卡在哪裡逼人看廣告/付費」的商業化節點。

## 📋 執行步驟

1. **核心行為與操作邊界（0-30秒）拆解**
   - **輸入/輸出映射**：精確記錄操作模式（點擊、長按、滑動），並測試「誤觸寬容度」（Hitbox 是否大於視覺圖像）。
   - **操作極限與並發處理**：支援多點觸控嗎？連續狂點能跳過動畫嗎？操作過快時，系統是將輸入放入佇列 (Queue) 還是直接忽略無效點擊？
   - **物理與極限規則**：若是物理類/挪車類/合成類，測試物件擠壓、重疊、掉落加速度的邊界，尋找可能卡住的邊緣案例 (Edge cases)。
   - **單次行動反饋**：每次有效操作伴隨的數值、狀態或隱藏量表（如 Combo 累積）變化。

2. **單局目標層次與留存鉤子（5-15分鐘）拆解**
   - **目標多層次設定**：區分「即時通關目標」（如移出特定車輛/動物）與「局內次要目標」（如收集地圖鑰匙、完成步數寶箱進度）。
   - **隨機性 (RNG) 控制機制**：盤面初始掉落或生成是「純隨機」還是「偽隨機」？系統是否會在即將面臨「死局」時，刻意給予關鍵牌/棋子？
   - **失敗定義與容錯率**：明確定義什麼情況觸發 Game Over（無解狀態、步數耗盡、時間到、空間滿），以及遊戲給予的 Undo（上一步）機制的取得成本。
   - **「再一局」誘因與心流**：失敗時的接關條件（看廣告、差一步的心理暗示），以及從過關/失敗到重啟下一局的「點擊次數與中斷時間」（越短心流越連貫）。
   - **解鎖與難度節奏**：第一個新機制在哪一分鐘出現？玩家的認知負擔何時達到頂點？

3. **逆向經濟與數值控制模型**
   - **數據錄製**：以 Markdown 表格記錄前 20 關（或前 60 分鐘）的玩家狀態、升級成本、金幣產出與體力消耗模型。
   - **步數/時間設計差值**：記錄「通關實際最低所需步數」與「系統給定步數」的差值，找出遊戲逼迫玩家使用道具的隱形數學門檻（例如：給定步數僅為完美通關步數的 1.1 倍）。
   - **動態難度調節 (DDA) 偵測**：刻意連續失敗 3 次或 5 次後，開局盤面、隨機掉落物或過關門檻是否會有暗中增益（系統放水機制）。

4. **廣告/內購 (IAP) 節點記錄**
   - **激勵/插屏/橫幅廣告**：記錄出現時機與倒數時間。
   - 記錄「破冰禮包」出現的心理卡點。

## 📦 對齊 Markdown 工業標準產線 (Golden Layout)
完成拆解後，請將資料彙整成 Markdown 檔案，並且：
1. **[必備欄位]** 開頭必須標註 > 分析基礎： 與 > 負責人：。
2. **[視覺化]** 只要提到系統跳轉或狀態機，**強制使用 Mermaid Flowchart (graph TD)** 繪製流程圖。
3. **[硬核標籤]** 積極發掘以下技術指標，並用醒目的格式標示：
   - 物理手感偏移 (如 Y 軸防擋偏移 +150px)
   - 螢幕實體佔位 (強制 1080x2340 觀測盲點)
   - 效能與防發熱 (0 Overdraw 偽 3D、Object Pooling)
   - 動態音高多巴胺駭客 (Pitch Shifting)
   - 惡意情緒循環 (例如死局強制凍結)
