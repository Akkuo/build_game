# 05. 主題與敘事包裝 (調用 Narrative Designer)

## 🎯 拆解目的
拆解休閒遊戲中的 Meta 故事與輕度包裝（如家裝、拯救、化妝解謎）如何緊繫玩家情感脈絡以推動長期留存。

## 📋 執行步驟

1. **主題與情感鉤子 (Hooks)**
   - 分析外部廣告買量素材與遊戲內實際主題的關聯性（例如：「背叛與復仇」、「拯救落魄母女」）。
   - 拆解這個故事引導如何無縫融入核心循環的「代幣產出/花費」機制。

2. **無文字環境敘事**
   - 場景如何只透過美術道具的切換來訴說狀態變化（例如：破舊的客廳 → 花費星星打掃 → 溫馨的房間）。

3. **角色情緒即時反饋**
   - 角色模型在畫面上方針對玩家操作做出的即時反應（如：連擊時歡呼、卡關時緊張流汗、過關的慶祝肢體語言）。

## 📦 對齊 Markdown 工業標準產線 (Golden Layout)
完成拆解後，請將資料彙整成 Markdown 檔案，並且：
1. **[必備欄位]** 開頭必須標註 > 分析基礎： 與 > 負責人：。
2. **[視覺化]** 只要提到系統跳轉或狀態機，**強制使用 Mermaid Flowchart (graph TD)** 繪製流程圖。
3. **[硬核標籤]** 積極發掘以下技術指標，並用醒目的格式標示：
   - 物理手感偏移 (如 Y 軸防擋偏移 +150px)
   - 螢幕實體佔位 (強制 1080x2340 觀測盲點)
   - 效能與防發熱 (0 Overdraw 偽 3D、Object Pooling)
   - 動態音高多巴胺駭客 (Pitch Shifting)
   - 惡意情緒循環 (例如死局強制凍結)
